## 2. 产品概述

### 2.1 产品背景
随着企业广泛采用 AI/MaaS 服务，AI 成本管理成为痛点：
- 多云环境导致成本分散
- 缺乏业务维度的成本归因
- 月末账单滞后，无法实时监控
- 缺少成本优化建议

### 2.2 产品目标
构建企业级 AI 成本管理平台，实现：
1. **透明化**: 让每一分 AI 支出都清晰可见
2. **可控制**: 预算管理 + 实时预警
3. **可优化**: 提供成本优化建议
4. **可分摊**: 支持多维度成本归因

### 2.3 目标用户

**主要用户**:
1. **CTO/技术 VP** - 总体成本把控
2. **工程经理** - 团队成本管理
3. **FinOps 团队** - 成本分摊与优化
4. **平台工程师** - 工具维护

**用户画像**:
- 中大型企业（100+ 员工）
- 月 AI 支出 $1,000 - $100,000
- 使用 2+ 个 MaaS 提供商
- 有成本分摊需求

### 3.2 详细功能需求

#### FR-001: API 代理层
**描述**: 拦截并转发 AI API 请求，记录用量数据

**验收标准**:
- [ ] 支持 OpenAI API 格式
- [ ] 支持 Azure OpenAI 格式
- [ ] 透明转发，不修改请求/响应（除必要 header）
- [ ] 延迟增加 < 50ms
- [ ] 支持流式响应 (SSE)

**输入**: HTTP Request  
**输出**: HTTP Response + 数据库记录

---

#### FR-002: 成本归因标签
**描述**: 允许用户通过 HTTP Header 标记请求的业务属性

**标签定义**:
| Header | 必填 | 说明 | 示例 |
|--------|------|------|------|
| X-Cost-Project | 是 | 项目名称 | customer-service-bot |
| X-Cost-Team | 否 | 团队名称 | platform-team |
| X-Cost-Env | 否 | 环境 | production/staging/dev |
| X-Cost-User | 否 | 用户ID | user-12345 |

**验收标准**:
- [ ] 支持上述所有标签
- [ ] 缺失必填标签时记录警告
- [ ] 标签值长度限制 128 字符
- [ ] 支持特殊字符过滤

---

#### FR-003: 成本计算引擎
**描述**: 根据 Token 用量和定价计算成本

**定价模型**:
```
成本 = (Input Tokens / 1000) * Input Price 
     + (Output Tokens / 1000) * Output Price
```

**支持的模型定价**:
- OpenAI: GPT-4, GPT-4 Turbo, GPT-3.5 Turbo, Embedding
- Azure OpenAI: 同上
- Anthropic: Claude 3 Opus, Sonnet, Haiku
- 国内模型: 文心一言、通义千问、智谱 GLM

**验收标准**:
- [ ] 准确率 100%
- [ ] 支持定价热更新
- [ ] 支持自定义定价
- [ ] 未知模型标记为 "unknown"

---

**描述**: 可视化展示成本数据

**页面布局**:

```
┌─────────────────────────────────────────────┐

└─────────────────────────────────────────────┘
```

**功能需求**:
- [ ] 支持时间范围筛选（24h/7d/30d/自定义）
- [ ] 支持项目/团队筛选
- [ ] 图表支持导出 PNG
- [ ] 表格支持排序和分页
- [ ] 响应式设计（PC/平板/手机）

---

#### FR-005: 预算管理
**描述**: 设置预算限额并监控使用情况

**预算类型**:
- 全局预算: 企业总预算
- 项目预算: 单个项目限额
- 团队预算: 单个团队限额

**预算周期**:
- 日预算
- 周预算
- 月预算
- 季度预算

**预警级别**:
| 阈值 | 动作 |
|------|------|
| 50% | 信息通知 |
| 80% | 警告通知 |
| 100% | 严重通知 + 可选阻断 |

**通知渠道**:
- Webhook (飞书/钉钉/Slack)
- 邮件
- 站内信

---

#### FR-006: REST API
**描述**: 提供数据查询接口

**端点列表**:
```
GET  /api/v1/health              # 健康检查
GET  /api/v1/stats/summary       # 汇总统计
GET  /api/v1/stats/projects      # 项目统计
GET  /api/v1/stats/teams         # 团队统计
GET  /api/v1/stats/daily         # 日报数据
GET  /api/v1/records             # 详细记录
POST /api/v1/budgets             # 创建预算
GET  /api/v1/budgets             # 查询预算
```

**通用参数**:
- `days`: 统计天数（默认 30）
- `start_date`: 开始日期
- `end_date`: 结束日期
- `project`: 项目筛选
- `team`: 团队筛选

**响应格式**:
```json
{
  "code": 0,
  "message": "success",
  "data": { ... }
}
```

## 4. 非功能需求

### 4.1 性能需求

| 指标 | 目标 | 说明 |
|------|------|------|
| API 响应时间 | P99 < 200ms | 代理层延迟 |
| 仪表盘加载 | < 3s | 首屏加载 |
| 数据库查询 | < 100ms | 常规查询 |
| 并发请求 | > 1000 RPS | 单实例 |

### 4.2 可用性需求

| 指标 | 目标 | 说明 |
|------|------|------|
| 系统可用性 | 99.9% | 年度停机 < 8.76h |
| 故障恢复 | < 5min | 自动重启 |
| 数据持久化 | 100% | 零数据丢失 |

### 4.3 安全需求

| 需求 | 级别 | 说明 |
|------|------|------|
| 数据传输加密 | 必须 | HTTPS/TLS 1.3 |
| API Key 存储 | 必须 | 加密存储 |
| 访问控制 | 必须 | RBAC |
| 审计日志 | 必须 | 记录所有操作 |
| SQL 注入防护 | 必须 | 参数化查询 |
| XSS 防护 | 必须 | 输入过滤 |

### 4.4 可扩展性需求

- 支持水平扩展（无状态设计）
- 支持多种数据库后端（SQLite/PostgreSQL/MySQL）
- 支持缓存层扩展（Redis）
- 插件化架构支持新厂商接入

## 6. 数据模型

### 6.1 实体关系图

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│   Project    │────<│ UsageRecord  │>────│   Provider   │
└──────────────┘     └──────────────┘     └──────────────┘
                            │
                            >
                     ┌──────────────┐
                     │    Budget    │
                     └──────────────┘
                            │
                            >
                     ┌──────────────┐
                     │ BudgetAlert  │
                     └──────────────┘
```

### 6.2 核心实体定义

**UsageRecord (使用记录)**:
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| request_id | VARCHAR(64) | 唯一请求ID |
| timestamp | DATETIME | 请求时间 |
| provider | VARCHAR(32) | 提供商 |
| model | VARCHAR(64) | 模型名称 |
| prompt_tokens | INT | 输入token |
| completion_tokens | INT | 输出token |
| cost_total | DECIMAL(10,6) | 总成本(USD) |
| project | VARCHAR(128) | 项目标签 |
| team | VARCHAR(64) | 团队标签 |

**Budget (预算)**:
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| name | VARCHAR(128) | 预算名称 |
| scope_type | VARCHAR(32) | 范围类型 |
| scope_value | VARCHAR(128) | 范围值 |
| amount | DECIMAL(10,2) | 预算金额 |
| period | VARCHAR(32) | 周期 |
| alert_thresholds | JSON | 预警阈值 |
